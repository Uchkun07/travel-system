- 你的安装路径：D:\ProgramFiles\Jmeter\bin

## 3. 脚本设计说明（可直接运行）

脚本采用两阶段：

1. 登录准备阶段（单线程）

- 先请求管理员登录 /api/admin/login
- 再请求用户登录 /api/user/login
- 将 token 存入全局属性，供压测线程组复用

2. 全接口压测阶段（高并发）

- 从 api_cases.tsv 逐条读取 method/path/authType/body
- 根据 authType 自动注入 Authorization
- 统一使用 HTTP JSON 请求

默认高压参数（已写入脚本）：

- threads: 400
- rampUp: 60 秒
- loops: 20

说明：

- 用例里路径参数已统一替换为 1（例如 /detail/{id} -> /detail/1）
- 为了保证脚本可直接执行，绝大多数写操作接口的 body 使用 {} 占位；这会导致部分接口返回 400/403，属于预期

## 6. 结果解读

1. 错误率高的主要原因

- 本次是全接口统一压测，很多管理端与用户端接口需要严格登录态、角色权限与业务入参
- 用户登录 token 未能稳定获取，用户态接口大量返回 401
- 管理接口中大量写接口使用了通用占位 body，触发 400/403

2. 为什么仍然有参考价值

- 脚本可直接运行、可重复执行
- 在一致条件下可持续对比 QPS 和平均响应时间变化
- 适合作为第二轮性能优化后的基线压测方案
